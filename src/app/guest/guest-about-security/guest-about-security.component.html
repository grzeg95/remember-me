1. General provisions:
    1. Sing in by Google or Facebook or try it anonymously.
    2. Each user is individually encrypted.
    3. No data from database is stored in browser.
    4. Delete the account and all your data if you want.
    5. There is no logic and there will be no logic for an admin to sing in as a different user.
    6. Feel free to ask me any questions.
  2. How it works:
    1. There are two independent accounts one for the Firebase and second one for the KMS. The database is not possible to decrypt without accessing to the KMS.
    2. After the account is created, a custom private/symmetric key claim in the JWT is created which is encrypted with the asymmetric key from the KMS.
    3. The user asks to decrypt the private key by the KMS.
    4. If it is possible, the user stores the private key in the IndexedDB as the CryptoKey.
    5. The CryptoKey is only used for decryption with no export option.
    6. The user has a random simple math test to determine the ownership of the stored CryptoKey.
    7. Only one CryptoKey is stored in the IndexedDB.
