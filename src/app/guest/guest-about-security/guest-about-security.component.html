User data encryption policy

  1. General rules.
    1. The user has only encrypted data in the database.
    2. No user data from the database is saved in the browser.
    3. It is not possible to read raw user data from the database in logical form.
    4. There is a place for metadata of recently logged in users, which can be managed using the option "Log in" -> "Users".
  2. For noobs.
    1. There are two projects one for Firebase and second one for asymmetric key.
    2. User has encrypted by asymmetric key individual symmetric key stored in the JWT token.
    3. If there is no CryptoKey stored in the IndexedDB associated with user then user will send request for symmetric key and then store it as a CryptoKey.
    4. CryptoKey is stored only for decrypting without option to extractable key from it.
    5. Guest can remove users CryptoKeys that was stored via 'Log in' -> 'Users' option.
