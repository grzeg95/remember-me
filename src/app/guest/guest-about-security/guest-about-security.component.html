User data encryption policy

  1. There are two projects one for Firebase and second one for asymmetric key.
  2. User can read own data from database.
  3. User data is encrypted.
  4. There is no possibility to read raw user data from the database in the logic form.
  5. User has encrypted by asymmetric key individual symmetric key stored in the JWT token.
  6. If there is no CryptoKey stored in the IndexedDB associated with user then user will send request for symmetric key and then store it as a CryptoKey.
  7. CryptoKey is stored only for decrypting without option to extractable key from it.
  8. Guest can remove users CryptoKeys that was stored via 'Log in' -> 'Users' option.
  9. No user data from database is stored in the browser.
